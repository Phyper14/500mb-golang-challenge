## Key decisions

- **Storage: Redis** (não Postgres/MariaDB/MySQL) — menor overhead de memória entre os 4 engines permitidos pelo desafio, e um único ZSET por device (score=`ts`, member=ponto binário) serve tanto a query por janela temporal (`ZRANGEARGS ... BYSCORE`) quanto a leitura dos últimos 256 pontos do anomaly (`ZRANGE -256 -1`), sem precisar de estrutura de dados separada.
- **Codec binário fixo (53 bytes/ponto)** em vez de JSON para os members do ZSET — elimina overhead de `encoding/json`/reflection no hot path de escrita (60% do mix de tráfego steady é `POST /telemetry`). Inclui nonce aleatório de 4 bytes para dois pontos com campos idênticos (incluindo `ts`) nunca colidirem como o mesmo member do ZSET.
- **`net/http` puro (stdlib, Go 1.22+ method+path routing)**, sem framework (gin/echo/chi) — menor footprint de binário/RSS possível, alinhado ao tema central do desafio (eficiência = 32% do score).
- **Bug de ponto flutuante encontrado via TDD**: no cálculo do z-score, somar magnitudes quase idênticas deixava `stddev` como um resíduo não-zero (~1e-15) em vez de exatamente zero, inflando `z = tiny/tiny` para ~1 espuriamente. Corrigido com um epsilon guard (`stddevEpsilon = 1e-9`) em `internal/domain/anomaly.go`.
- **Retenção limitada por device** (`MAX_POINTS_PER_DEVICE`, default 5000, trimming automático dos pontos mais antigos) — evita crescimento ilimitado de memória no Redis sob carga prolongada (cenário `endurance`/`capacity` do benchmark).
- **Imagem `distroless/static-debian12:nonroot`** com binário `CGO_ENABLED=0` — sem shell, sem libc, non-root por padrão (uid 65532), ~12.3MB. Satisfaz hardening do desafio (`read_only`, `cap_drop: [ALL]`, non-root) sem esforço extra.
- **Rede Docker dividida em `backend` (internal: true) + `edge`** — o LB precisa estar em ambas: `internal: true` sozinho bloqueia o port publishing do host (`8080:80` nunca ficava acessível externamente). Bug encontrado e corrigido durante o teste real com `docker compose up`.
- **`.gitignore` tinha `api` genérico** que acabou casando com o diretório `cmd/api/`, fazendo `main.go` nunca ser rastreado pelo git. Corrigido para `/api` (âncora na raiz, só ignora o binário compilado ali). Bug encontrado testando o `setup-branches.sh` em cópias temporárias (3 rodadas de teste) antes de considerar o script pronto.
- **`scripts/setup-branches.sh`** deriva a branch `implementation` a partir de `main` via allowlist (`docker-compose.yml`, `deploy/`, `me.json`) — testado e confirmado que produz exatamente os arquivos exigidos pela regra de submissão, sem sobra.
- **zap (não logrus)** para logging estruturado — menor overhead de alocação, padrão de baixo custo em runtime.
- **X-Instance-Id em toda resposta**, incluindo `/metrics` (que usa `promhttp.HandlerFor` por padrão não inclui o header — precisou de wrapper explícito `handleMetrics`).
